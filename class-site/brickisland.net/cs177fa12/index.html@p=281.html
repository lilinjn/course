    If you are working on OSX, here are some tips for how to get the code compiling. I was able to use Homebrew (http://mxcl.github.com/homebrew/) to install SuiteSparse and all the dependencies using

    $ brew install suite-sparse

    This also installs METIS and a few other libraries and links them into brew’s prefix path (on my machine, the default /usr/local). The makefile variables below seem to do the trick:

    # Mac OS X
    DDG_INCLUDE_PATH = -I/usr/local/include
    DDG_LIBRARY_PATH = -L/usr/local/lib
    DDG_BLAS_LIBS = -framework Accelerate
    DDG_SUITESPARSE_LIBS = -lspqr -lcholmod -lmetis -lcolamd -lccolamd -lcamd -lamd -lm -lsuitesparseconfig -ltbb
    DDG_OPENGL_LIBS = -framework OpenGL -framework GLUT

    Note the addition of -lsuitesparseconfig -ltbb to DDG_SUITESPARSE_LIBS to make the ddg code compile.

    A couple details about compiling the code that came up during our session on Thursday:

    1. If you don’t already have a local include directory, you need to create it by typing sudo mkdir /usr/local/include before installing SuiteSparse. Otherwise, SuiteSparse installation will fail when you type make install (it will just copy a bunch of files into a new file called /usr/local/include, rather than creating the appropriate directory!).

    2. SuiteSparse will compile METIS for you, but only if the METIS directory is inside the SuiteSparse directory, and only if it is named metis-4.0 (not metis-4.0.3 or whatever you get by default).

    3. In the latest version of SuiteSparse, the configuration file can be found in the SuiteSparse_config directory, not UFconfig as described in the libddg docs.

    4. Finally, you need to add -lsuitesparseconfig to the list of libraries in the Makefile for the assignment code (this library did not exist in previous versions of SuiteSparse)

    Hope that helps!
